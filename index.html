Introduction

Markdown is a lightweight markup language for creating formatted text using a plain-text editor. It was created by John Gruber and Aaron Swartz in 2004. It was created as a markup language that is appelaing for human readers in its source code form. Its key design goal was readability, that the language is readable as is, without looking like it has been marked up with formatting instructions, unlike the much heavier markup languages like the RTF (Rich Text Formatting) or HTML(Hyper Text Markup Language).

You can add Markdown formatting elements to a plaintext file using a text editor application. Or you can use one of the many Markdown applications for macOS, Windows, Linux, iOS, and Android operating systems. There are also several webbased applications specifically designed for writing in Markdown. Depending on the application you use, you may not be able to preview the formatted document in real time. But that’s okay. According to Gruber, Markdown syntax is designed to be readable and unobtrusive, so the text in Markdown files can be read even if it isn’t rendered.

The overriding design goal for Markdown’s formatting syntax is to make it as readable as possible. The idea is that a Markdown-formatted document should be publishable as-is, as plain text, without looking like it’s been marked up with tags or formatting instructions.

How Markdown Works

When you write in Markdown, it is stored as a plain text file saved with the .md or .markdown extension. Then you'll need a Markdown Application that will convert the Markdown-formatted text to HTML so that it can be displayed in web browsers.

Markdown applications use a Markdown processor(a parser or an implementation) to take Markdown-formatted text and output it as HTML.

Using Markdown

Markdown is one of the most popular markup languages in the world. It can be used to do a variety of things! The following are some of the very popular uses of Markdown:

  • Websites: There are a wide range of tools available to use Markdown in your websitea. If you want to use a CMS (Content Management System) to power your site, popular CMS like Ghost & Wordpress support Markdown.
  • Documents: You can use Markdown to write effectively formatted documents just as you would on any word processors except that you won't have all the sophisticated formatting features like shadows and 3D text.
  • Notes: Markdown is great for notes. Markdown lets you write notes in your Text/Code editor.
  • Documentation: Markdown is a natural fit for technical documentation. Companies like GitHub are increasingly switching to Markdown for their documentation.
  • Books:Markdown is great for writing books. It is readable even in source code form making it easier for you to see the content and the formatting at the same time.
Basic Syntax

Headings

To create a heading, add number signs (#) in front of the word or the phrase. The numebr of # signs correspond to the heading level.

----------
# Level 1 Heading
## Level 2 Heading
...
###### Level 6 Heading
----------

The above headings output exactly as their HTML equivalents. And you can even use the html tags to do the same in Markdown.

Paragraphs

To create a paragraph, you just write text. To seperate 2 paragraphs, leave an empty line between them. You shouldn't indent a paragraph.

----------
This is a paragraph

This is another paragraph!
----------

Line Breaks

To create a line break, use 2 or more spaces and then use return/enter.

Bold Text

In order to use bold text, enclose the word or phrase in two asterisks or underscores on both sides.

----------
This is **bold** text
This is __bold text too__
----------

Italic text

Enclose the text in single asterisks or underscores to create a italicized text

----------
Some *italic* text
More _italic text_
----------

Bold and Italic

To emphasize text with both italics and bold at the same time, you can enclose the text in three underscores or asterisks.

----------
***Some Text***
___Some Text___
_**This works too**_
__*So will this.*__
----------

Blockquotes

To enter text as a blockquote, add > in front of the text.

-----------
> Remember! You are awesome!
-----------
Lists

Ordered Lists

To create an ordered list, add line items with numbers followed by periods.

-----------
1. Rock the world
2. Jump from a plane
3. Go to gym
-----------

Nesting lists

To nest lines in an ordered list, indent the items four spaces or one tab.

-----------
1. Task 1
2. Task 2
    1. Task 2a
    2. Task 2b
3. Task 3
-----------

Unordered List

To create an unordered list, add dashes (-), asterisks (*), or plus signs (+) in front of line items.

-----------
- First item
* Second item
+ Third item
-----------
Links and Images

Links

To use links for words/phrases in your document, enclose it in square brackets, followed by parantheses in you must include the URL.

-----------
Click [here](https://bio.link/charanmanikanta)
-----------

Images

The syntax to include an image and a link is very similar. You can insert an image by writing an exclamation mark followed by square brackets, and parantheses containing the location URL of the image. You can optionally put text describing the image, inside the square brackets.

-----------
![Image of a fluffy orange cat](https://www.mywebsite.com/fluffy-orange-cat.png)
-----------
Code

Enclode the text between backticks to format it to look like code.

------------
The `array.splice()` is a very useful function in JavaScript
------------
Horizontal Rules

To create a horizontal rule, use three or more asterisks (***), dashes (---), or underscores (___) on a line by themselves.

------------
***
---
________
------------
